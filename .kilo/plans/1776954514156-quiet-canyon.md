# Plan: Fix Sign-In Flow Issues

## Problem Analysis

### Issue 1: Stuck at Loading Screen
**Root Cause:** The loading screen issue is likely caused by the `onLoginUser` function in `src/actions/auth/index.ts`.

**Current Flow:**
1. User visits `/dashboard` → `(dashboard)/layout.tsx` calls `onLoginUser()`
2. `onLoginUser()` calls `currentUser()` from Clerk
3. If no Clerk user → `redirectToSignIn()` is called (server-side redirect)
4. If Clerk user exists but NOT in Prisma DB → returns `{ status: 400 }` (truthy value)
5. The layout checks `if (!authenticated)` but `{ status: 400 }` is truthy, so it doesn't redirect
6. However, `authenticated.domain` is `undefined`, causing downstream rendering issues

**Key Issues:**
- Line 14 in `(dashboard)/layout.tsx`: `if (!authenticated) redirect('/auth/sign-in')` - doesn't handle the case where `authenticated` is `{ status: 400 }`
- Line 57 in `actions/auth/index.ts`: Returns `{ status: 400 }` instead of redirecting to sign-in or returning `null`
- No loading state in the dashboard layout while checking authentication

### Issue 2: Homepage Buttons Not Opening Sign-In Page
**Current Button Behavior:**
1. **"Free Trial" button** (`navbar/index.tsx:29-34`): Links to `/dashboard`
   - This should trigger auth guard and redirect to sign-in
   - But if user is not in DB, it gets stuck (see Issue 1)

2. **"Start For Free" button** (`page.tsx:50-55`): Links to `/auth/sign-up`
   - This is correct behavior (sign-up page)

3. **"Get Started" buttons** (`page.tsx:104-109`): Link to `/dashboard?plan=${card.title}`
   - Same issue as "Free Trial"

**Actual Problem:** The buttons DO navigate, but the auth flow gets stuck because:
- User clicks button → goes to `/dashboard` → layout checks auth → if not fully registered, gets stuck with no proper redirect

### Step 1: Fix `onLoginUser` in `src/actions/auth/index.ts`
**File:** `src/actions/auth/index.ts:37-59`

**Changes:**
- If user exists in Clerk but NOT in Prisma DB → should redirect to sign-up or return a proper value
- Ensure consistent return values (either user object or redirect, not mixed)

```typescript
export const onLoginUser = async () => {
  const user = await currentUser()
  if (!user) return redirectToSignIn()
  
  try {
    const authenticated = await client.user.findUnique({
      where: {
        clerkId: user.id,
      },
      select: {
        fullname: true,
        id: true,
        type: true,
      },
    })
    
    if (!authenticated) {
      // User exists in Clerk but not in our DB
      // Redirect to complete registration or sign-in
      return redirectToSignIn()
    }
    
    const domains = await onGetAllAccountDomains()
    return { status: 200, user: authenticated, domain: domains?.domains }
  } catch (error) {
    // Log error properly
    console.error('Login error:', error)
    return redirectToSignIn()
  }
}
```

const OwnerLayout = async ({ children }: Props) => {
  const authenticated = await onLoginUser()
  
  // Check if authenticated is a redirect (from redirectToSignIn)
